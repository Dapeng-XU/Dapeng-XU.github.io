Part 2: General Problem about Canvas 2D

2.1 What functions do the canvas 2D API contains?

For generality, all the functions called in this projected are well supported by lots of modern browsers, and another condition these functions satisfied is that they are described by some web standard. The former condition keeps the compatibility, the last condition condition confirms the function will have been supported by lots of platform for a long term.

The sets of this kind of functions in canvas rendering 2D context are[1]: drawing rectangles, drawing text, line styles, text styles, fill and stroke styles, gradients and patterns, shadows, paths, drawing paths, compositing, the canvas state. Note that all functions listed will not be used in this project.

2.2 How do we deal with the canvas resizing correctly?

If we consider the canvas elements as a sort of normal HTML element, we will make a mistake. We should understand canvas a little more. That is to say, there is a buffer attached to each canvas element in the memory. Every time we draw something on the canvas, it will first modify the buffer, then the buffer will be drawn on the canvas element. Therefore, when a canvas resizing, the default manner is resize the canvas element, but keep the size of the buffer same as ever. The key is we should manually update the size of buffer.

It is implemented in dpConvas2D.setPosition(). If you would like to get more details, see [2].

2.3 Is there any library you used? And what are they?

This project is so important that it is the final project of the class computer graphics, of which the contents are really interesting.
